# 预测性维护平台用户操作手册

## 1. 文档说明

本文档依据当前系统代码逻辑整理，面向实际使用人员，按 Web 端和平板端分别说明操作方法，并给出一个完整项目从创建到执行、再到报告输出的标准流程。

系统当前的业务分工如下：

1. Web 端负责基础数据维护、项目创建、任务生成、任务下发、结果查看和报告输出。
2. 平板端负责任务下载、离线执行、拍照留证和离线结果上传。
3. 离线执行不是独立业务线，而是在线任务执行链路中的一个环节。通常应先在 Web 端完成项目和任务准备，再在平板端离线执行。

## 2. 角色分工建议

为保证流程顺畅，建议按以下职责分工使用系统：

1. 管理员：维护用户、角色、权限。
2. 方案工程师：维护巡检方案、模板和检测项。
3. 项目经理：创建项目、选择设备、匹配方案、确认项目内容。
4. 执行工程师：下载任务、现场执行、补充照片和备注、上传结果。
5. 审核或管理人员：查看任务结果、查看项目报告、导出 Word 报告。

## 3. 执行一个项目的完整流程

以下是系统当前代码逻辑对应的标准执行顺序，建议严格按顺序操作。

### 3.1 总流程

1. 先准备基础资料：客户、设备、巡检方案、执行人员。
2. 再新建项目：填写基本信息、选择设备、匹配方案、调整方案、确认方案。
3. 系统自动生成项目关联设备、产品、巡检任务和任务项。
4. 在任务列表中按项目筛选任务。
5. 在线执行时，可直接打开任务查看方案和结果；离线执行时，先下载任务到平板。
6. 工程师在平板端执行任务，填写结果、照片和备注。
7. 网络恢复后上传离线结果。
8. 管理人员在 Web 端查看项目报告，补充总结并导出 Word。

### 3.2 推荐先后顺序

1. 先建客户。
2. 再建设备。
3. 再建巡检方案。
4. 再检查用户和角色是否已配置。
5. 然后创建项目。
6. 创建项目后再下载任务。
7. 下载任务后再执行离线巡检。
8. 离线巡检完成后再上传数据。
9. 所有任务完成后再查看或导出项目报告。

## 4. Web 端操作手册

## 4.1 登录系统

1. 打开系统登录页。
2. 使用已分配账号登录。
3. 登录成功后，系统会根据角色权限显示可访问菜单。
4. 如果没有某个菜单，说明当前账号没有对应权限，需要管理员分配角色或权限。

## 4.2 基础数据准备

在创建项目前，请先确认以下基础数据已经准备完成。

### 4.2.1 客户维护

1. 进入“客户管理”。
2. 新增客户信息。
3. 至少确认客户名称准确无误。
4. 若后续项目需要关联客户联系人，请一并维护完整。

### 4.2.2 设备维护

1. 进入“设备管理”。
2. 新增设备。
3. 维护设备所属客户、型号、数量、工厂、车间、电气室等信息。
4. 如果一个项目后续要生成多个设备任务，请确保设备数量填写正确，因为系统会据此补齐对应产品并生成任务。

### 4.2.3 巡检方案维护

1. 进入“方案维护”。
2. 新建或编辑巡检方案。
3. 配置模板名称、检测类型和检测项。
4. 检测项内建议完整配置分类、排序、操作指导、显示条件、推荐规则、隐患内容和维修建议。
5. 发布前先确认模板内容完整，因为项目确认后系统会基于模板自动生成任务项。

### 4.2.4 用户与角色维护

1. 管理员进入“用户管理”和“角色管理”。
2. 创建用户并分配角色。
3. 确保项目经理至少具备项目和任务查看权限。
4. 确保执行工程师至少具备在线任务和离线任务权限。

## 4.3 Web 端创建项目

项目创建是整个流程的核心入口，系统采用五步向导。

### 4.3.1 第一步：填写基本信息

1. 进入“项目列表”。
2. 点击“新建项目”。
3. 在“基本信息”步骤中填写以下内容：
4. 项目名称。
5. 服务编号。
6. 客户。
7. 工厂。
8. 城市。
9. 客户联系人。
10. 项目经理。
11. 主负责工程师。
12. 执行工程师。
13. 维护方案标识信息。
14. 填写完成后进入下一步。

说明：

1. 项目名称和客户必须有效。
2. 项目保存时，系统会写入项目基础信息。
3. 新建项目时，系统还会自动初始化一条项目报告记录。

### 4.3.2 第二步：选择设备

1. 在“选择设备”步骤中添加本项目涉及的设备。
2. 可以逐台添加，也可以按界面支持的批量方式导入。
3. 对每台设备补充型号、数量、工厂、车间、电气室等信息。
4. 如果现场存在外围检测需求，也要在这里把相关对象准备好，供后续匹配外围方案。

说明：

1. 设备数量会影响后续生成的产品数量。
2. 项目保存时，系统会先确保设备对应的产品记录存在，再生成任务。

### 4.3.3 第三步：匹配方案

1. 在“匹配方案”步骤中，系统会根据设备分类、子分类和型号推荐合适模板。
2. 为每种设备型号选择对应的巡检方案。
3. 若存在外围检测项，也要为外围检测选择对应模板。
4. 若系统未自动匹配成功，请手动选择正确模板。

### 4.3.4 第四步：调整方案

1. 进入“调整方案”步骤。
2. 检查系统已匹配的检测项树。
3. 按项目实际情况勾选需要保留的检测项。
4. 如有不适用项，可在此阶段调整。
5. 确认后进入下一步。

### 4.3.5 第五步：确认方案

1. 进入“确认方案”步骤。
2. 再次确认项目基本信息、设备信息和方案信息。
3. 点击提交。
4. 系统开始保存项目。
5. 系统保存项目后，会自动执行以下动作：
6. 保存项目基础信息。
7. 保存项目与方案的匹配选择。
8. 同步项目设备与产品数据。
9. 建立项目设备与模板的关联关系。
10. 为设备检测自动生成巡检任务。
11. 为外围检测自动生成巡检任务。
12. 根据模板检测项自动生成任务项。
13. 初始化项目报告记录。
14. 提交完成后返回“项目列表”。

## 4.4 Web 端项目执行前检查

创建项目后，不要马上去现场执行，建议先做一次项目检查。

1. 回到“项目列表”。
2. 确认新项目已出现在列表中。
3. 确认项目状态正常。
4. 进入“任务列表-在线版”。
5. 按项目筛选任务。
6. 检查是否已生成设备检测任务和外围检测任务。
7. 检查任务编号、模板名称、设备型号、序列号信息是否合理。
8. 如任务数量异常，优先检查项目设备数量和模板匹配是否正确。

## 4.5 Web 端在线任务操作

### 4.5.1 查看在线任务

1. 进入“任务列表-在线版”。
2. 先选择项目。
3. 系统会加载该项目下所有巡检任务。
4. 可按关键词搜索任务编号、设备型号或方案名称。
5. 可查看每个任务当前状态。

### 4.5.2 下载任务到平板或本地

1. 在“任务列表-在线版”中勾选要下载的任务。
2. 点击“下载任务”。
3. 系统会将任务详情、任务项和模板快照写入本地离线库。
4. 下载成功后，该任务会在本地形成离线任务包。
5. 如果任务以前已经下载过，系统会先提示是否继续下载。

说明：

1. 下载任务后，系统会把在线任务状态更新为“进行中”。
2. 离线任务页只显示已经下载到本地的任务。

### 4.5.3 在线查看方案和结果

1. 在在线任务列表中可查看任务关联方案。
2. 任务执行完成后，可进入任务审核或结果查看页面检查数据。
3. 若实际执行使用平板离线完成，则结果上传后再回 Web 端查看更合适。

## 4.6 Web 端查看项目报告

1. 回到“项目列表”。
2. 点击“查看报告”。
3. 系统会按项目读取项目、客户、设备、任务、结果和附件信息。
4. 在报告页面中查看目录、设备信息、问题项、建议和附录图片。
5. 可在报告页面补充总结说明和备件建议。
6. 页面会自动保存总结草稿。
7. 确认内容无误后导出 Word 报告。

## 5. 平板端操作手册

平板端主要用于现场执行任务，重点是离线能力。

## 5.1 平板端标准使用流程

1. 先联网登录系统。
2. 进入“任务列表-在线版”。
3. 选择项目并下载任务。
4. 确认任务已经出现在“任务列表-离线版”。
5. 到现场后即使断网，也可以从“任务列表-离线版”执行任务。
6. 执行完成后，在有网络时返回“任务列表-离线版”上传数据。
7. 上传成功后，再回 Web 端查看结果和报告。

## 5.2 平板端下载离线任务

1. 打开平板端应用。
2. 进入“任务列表-在线版”。
3. 先选择项目。
4. 勾选需要离线执行的任务。
5. 点击“下载任务”。
6. 等待系统提示下载成功。
7. 再进入“任务列表-离线版”。
8. 确认任务已经存在。

建议：

1. 出发前完成全部任务下载。
2. 到现场前先抽查几个任务能否正常打开。
3. 如任务包不完整，应在联网状态下重新下载。

## 5.3 平板端离线执行任务

### 5.3.1 进入离线任务列表

1. 进入“任务列表-离线版”。
2. 页面会显示本地任务数、待同步数、可上传任务数和已勾选数。
3. 可按项目和关键词筛选本地任务。
4. 点击某条任务的“执行任务”。

### 5.3.2 打开任务后先补全任务头信息

进入“数据采集”页面后，先填写头部基础信息。

1. 设备检测任务必须填写或确认序列号。
2. 外围检测任务必须填写或确认电气室。
3. 所有任务都必须填写检查人员。
4. 设备检测任务建议同时确认设备名称、设备号、所属部门。

说明：

1. 这些信息会随任务结果一起保存到本地。
2. 上传前系统会校验关键字段是否已填写。
3. 设备检测任务缺少序列号或检查人员时不能上传。
4. 外围检测任务缺少电气室或检查人员时不能上传。

### 5.3.3 执行每个检测模块

1. 左侧为任务导航和检测模块列表。
2. 右侧为当前模块的检测项。
3. 按模块顺序逐项填写采集值。
4. 系统会依据模板规则自动判断正常或异常。
5. 若现场需要人工补充判断，请结合备注填写说明。
6. 如存在隐患或整改建议，应按实际情况补充记录。

### 5.3.4 添加现场照片

1. 每个检测项都可以添加照片。
2. 可直接拍照，也可从文件选择。
3. 每个检测项最多上传 3 张照片。
4. 照片保存后会写入本地离线库。
5. 可修改照片名称。
6. 如拍错了，可删除后重新上传。

### 5.3.5 保存与离开

1. 页面在操作过程中会自动保存草稿。
2. 切换页面、切到后台或关闭前，系统也会尽量保存本地草稿。
3. 即使暂时未上传，只要本地数据还在，就可以回到离线任务继续执行。

## 5.4 平板端上传离线结果

1. 回到“任务列表-离线版”。
2. 点击“刷新本地任务”，确认最新本地状态。
3. 勾选要上传的任务。
4. 点击“上传数据”。
5. 系统会先校验任务头信息是否完整。
6. 校验通过后，系统会逐个上传任务结果。
7. 上传时会同步以下内容：
8. 任务主信息。
9. 任务项结果。
10. 检查人员、序列号、电气室等补充信息。
11. 附件照片。
12. 上传成功后，本地任务会标记为已上传。
13. 之后可以在 Web 端继续查看结果和报告。

说明：

1. 如果所选任务没有待上传变更，系统会提示无需上传。
2. 如果某个任务上传失败，应优先检查网络、任务头信息和附件是否完整。

## 6. 按业务场景拆解的标准操作流程

## 6.1 场景一：新项目首次执行

1. 管理员先确认用户和权限。
2. 业务人员先建客户。
3. 业务人员再建设备。
4. 方案工程师建立巡检方案。
5. 项目经理新建项目。
6. 在项目向导中完成设备选择和方案匹配。
7. 提交项目，等待系统自动生成任务。
8. 在在线任务列表中按项目检查任务。
9. 使用平板下载任务。
10. 到现场执行离线巡检。
11. 回网后上传任务数据。
12. 回到 Web 端查看项目报告并导出。

## 6.2 场景二：项目方案调整后重新执行

1. 进入项目编辑。
2. 修改设备、模板或方案选择。
3. 再次提交项目。
4. 系统会更新项目设备关系，并按当前绑定关系重新同步任务。
5. 回到在线任务列表确认最新任务。
6. 重新下载需要执行的任务。
7. 在平板端执行更新后的任务。
8. 上传结果后重新查看报告。

## 6.3 场景三：仅执行离线巡检

1. 先在联网环境下载全部任务。
2. 到现场进入离线任务列表。
3. 逐个打开任务执行。
4. 补齐检查人员和设备信息。
5. 录入检测值并补充照片。
6. 全部任务完成后返回可联网环境。
7. 进入离线任务列表勾选任务并上传。
8. 回 Web 端复核结果。

## 7. 常见注意事项

1. 未先建设备和方案，项目创建后通常无法形成完整任务。
2. 项目创建完成后，应先检查任务是否已生成，再安排现场执行。
3. 平板离线执行前必须先下载任务，否则离线列表不会出现任务。
4. 设备检测任务上传前必须有序列号和检查人员。
5. 外围检测任务上传前必须有电气室和检查人员。
6. 单个检测项照片上限为 3 张。
7. 任务执行过程中建议边做边检查是否已保存，避免一次性录入过多。
8. 报告导出前建议在 Web 端先补齐总结说明和备件建议。

## 8. 给一线用户的最简执行口诀

1. 先建客户。
2. 再建设备。
3. 再建方案。
4. 然后建项目。
5. 项目确认后查任务。
6. 平板先下载再出发。
7. 现场离线执行并拍照。
8. 回网后上传数据。
9. 最后在 Web 端看报告、导出报告。
