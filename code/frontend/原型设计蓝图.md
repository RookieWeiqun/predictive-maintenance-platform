## 1. 项目概述

该项目名称为预防性维护报告平台，主要用于工程师在自动化与驱动产品的预防性维护项目中，收集设备维护数据，并生成预防性维护报告。

主要角色有项目经理，维护工程师，管理员

* 项目经理负责创建项目，导入设备数据，选择维护方案，分配维护任务等工作
* 维护工程师收到项目经理分配的维护任务后，在移动设备（平板电脑）上同步维护任务（包括采集现场照片），进入车间后可以离线录入设备数据并本地保存数据；离开车间现场后可以同步维护数据到云端并生成维护报告
* 维护工程师还可以将生成的维护报告导出为word版本并编辑，然后把生成PDF文档上传到平台中。
* 项目经理可以关闭和维护项目信息。
* 管理员可以管理用户信息、客户信息和设备维护标准方案等基础数据

## 2. 页面功能

页面功能包括两大部分：项目经理的项目管理界面和维护工程师的数据维护界面，两个界面相互关联，建议考虑模块化进行一定的资源共享。

2.1 项目管理界面

该界面主要项目经理使用，包括以下几个页面：

* 项目列表：包括项目浏览、搜索、新建项目、查看详情、修改和关闭项目等操作
* 新建项目：新建项目导航页，填写项目信息，导入维护设备，选择维护方案，分配执行资源等

  1，采用IxWorkflowSteps组件在顶部表示流程进度：基本信息->选择设备->匹配方案->调整方案->确认方案
  2，基本信息录入包括：项目名称，客户名称，工厂信息，项目经理，主任工程师等
  3，选择方案录入信息包括：选择维护产品大类，产品型号子类，维护方案（检测项目）
  4，调整方案根据上一步选择的方案进行调整，如删除或增加检测项目等
  5，最后一步上传设备清单，型号及数量等信息，并根据方案和清单生成维护任务，并分派具体工程师
* 项目详情：查看和修改项目详细信息，PDF报告导入等
* 报告详情：查看设备维护数据和报告预览，报告导出等
* 报告导出：导出预览，word版本报告下载

2.2 数据维护界面

该界面主要是维护工程师使用，包括一下几个页面：

* 执行任务列表：应该也是以项目为单位，每个项目包含多个设备维护任务
  * 一类设备是同样的方案，可以根据这个方案为每个设备创建报告模板，基于这个模板填写报告数据
  * 维护方案调整：每个设备可以单独调整维护方案，如基于标准方案的减项，手动增加新检测项目等
* 报告数据采集：以设备为单位，按照维护方案收集设备相关的各类数据：
  * 基本公共信息：如环境数据，车间数据，柜体情况等
  * 设备信息：序列号、物理参数及电气参数等
* 报告数据查看：数据收集完毕后，工程师可以再次查看和调整数据项目，并可以手动同步数据到云端

## 3. 主要对象定义及数据结构

本项目涉及的主要对象：

* 维护项目：项目编号，项目名称，客户名称，工厂信息，项目状态，项目经理，主任工程师，执行工程师列表，设备清单
* 维护任务：任务类型（外围检测和设备检测），设备信息，维护状态，维护工程师，维护时间，维护方案等
* 设备信息：设备类型，设备型号，设备序列号
* 标准方案：任务类型，设备类型，检测项目树（根据不同项目分级）
* 客户信息：客户名称，客户联系人，联系方式等
* 报告信息：以项目为单位出具报告，包括项目信息，最终维护方案，各类维护任务的完成数据，根据数据进行评估结果，维护结论和维护建议，备件建议等
* 用户管理：用户信息的增删改查。

## 4. 主要技术规范

样式设计规范：

* 使用西门子ix组件和图标样式规范进行原型设计
* 组件设计文档参考：[Components overview | Siemens Industrial Experience](https://ix.siemens.io/docs/components/overview)
* 图标设计文档参考：[Icon library | Siemens Industrial Experience](https://ix.siemens.io/docs/icons/icon-library)
* 注意样式设计采用统一变量，满足如深浅主题切换等需求

原型假数据设计规范

* 原型假数据请统一管理在一个mockdata的文件夹中，按模块分类，使用json文件格式
* 一些枚举变量等公共数据，请统一放到mockdata/common/文件夹中
* 需要考虑后端接口设计的要求
