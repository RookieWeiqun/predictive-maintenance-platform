# 预防性维护报告平台 - 原型设计方案

## 一、设计思路概述

### 1.1 设计原则
- **角色分离**：根据项目经理、维护工程师、管理员三个角色，设计不同的界面入口和功能模块
- **模块化设计**：共享通用组件和布局，提高代码复用性
- **响应式适配**：支持桌面端（项目经理/管理员）和移动端（维护工程师平板）
- **离线优先**：维护工程师界面支持离线数据采集和本地存储
- **渐进式开发**：采用敏捷开发思路，逐步完善每个页面

### 1.2 技术架构（简化版）
- **框架**：Vue 3 + TypeScript + Vite
- **UI组件库**：Siemens iX Design System（仅使用基础组件）
- **路由管理**：Vue Router（简单路由配置）
- **数据管理**：组件内使用 ref/reactive，暂不引入状态管理库
- **数据模拟**：页面内直接定义 Mock 数据，简单实用

### 2.2 角色视图切换策略

**方案A：统一入口 + 动态菜单（推荐）**
- 所有角色使用同一个应用框架
- 根据用户角色动态渲染不同的导航菜单
- 通过路由守卫控制页面访问权限

**方案B：独立应用入口**
- 为不同角色创建独立的入口页面
- 每个角色有独立的布局和导航结构

**建议采用方案A**，便于代码复用和维护。

## 三、页面模块划分

### 3.1 项目管理界面模块（项目经理）

#### 3.1.1 项目列表页 (`/project/list`)
**功能要点：**
- 项目卡片/列表展示（项目编号、名称、客户、状态、进度）
- 搜索和筛选（按状态、客户、时间范围）
- 新建项目按钮
- 批量操作（关闭、导出）
- 分页/虚拟滚动

**主要组件：**
- `IxCard` - 项目卡片
- `IxInput` - 搜索框
- `IxSelect` - 筛选下拉
- `IxButton` - 操作按钮
- `IxTable` - 列表视图（可选）

#### 3.1.2 新建项目页 (`/project/create`)
**功能要点：**
- 多步骤向导流程（使用 `IxWizard` 或 `IxStepper`）
  - 步骤1：项目基本信息
  - 步骤2：导入设备清单（支持Excel导入）
  - 步骤3：选择维护方案（基于设备类型）
  - 步骤4：分配执行资源（选择维护工程师）
  - 步骤5：确认并创建
- 表单验证
- 保存草稿功能

**主要组件：**
- `IxWizard` / `IxStepper` - 步骤导航
- `IxForm` + `IxFormGroup` - 表单
- `IxInput`, `IxSelect`, `IxTextarea` - 表单控件
- `IxUpload` - 文件上传
- `IxTree` - 方案选择树形结构

#### 3.1.3 项目详情页 (`/project/detail/:id`)
**功能要点：**
- 项目信息卡片（可编辑）
- 设备清单表格
- 维护任务列表
- 报告状态和预览
- PDF报告上传
- 项目操作（关闭、修改、导出）

**主要组件：**
- `IxCard` - 信息卡片
- `IxTable` - 数据表格
- `IxTabs` - 标签页切换（设备/任务/报告）
- `IxModal` - PDF上传弹窗

#### 3.1.4 报告详情页 (`/project/report/:id`)
**功能要点：**
- 报告预览（PDF/HTML）
- 报告数据展示（设备数据、评估结果、建议）
- 报告导出（Word下载）
- 报告编辑历史

**主要组件：**
- `IxCard` - 报告卡片
- `IxTabs` - 报告章节切换
- `IxButton` - 导出按钮
- 自定义PDF预览组件

### 3.2 数据维护界面模块（维护工程师）

#### 3.2.1 执行任务列表页 (`/task/list`)
**功能要点：**
- 按项目分组显示任务
- 任务状态标识（待执行/进行中/已完成/已同步）
- 离线/在线状态指示
- 任务筛选和搜索
- 下拉刷新/同步数据

**主要组件：**
- `IxCard` - 项目卡片
- `IxAccordion` - 可折叠项目列表
- `IxBadge` - 状态标签
- `IxButton` - 同步按钮

#### 3.2.2 维护方案调整页 (`/task/scheme/:taskId`)
**功能要点：**
- 显示标准方案树形结构
- 支持勾选/取消检测项目（减项）
- 支持手动添加自定义检测项目（增项）
- 方案预览和确认

**主要组件：**
- `IxTree` - 方案树
- `IxCheckbox` - 项目选择
- `IxInput` - 自定义项目输入
- `IxButton` - 保存按钮

#### 3.2.3 报告数据采集页 (`/task/collect/:taskId`)
**功能要点：**
- 多步骤数据采集向导
  - 步骤1：设备基本信息
  - 步骤2：环境数据（温度、湿度等）
  - 步骤3：电气参数（电压、电流等）
  - 步骤4：检测项目数据录入
  - 步骤5：现场照片采集
- 离线数据本地存储（IndexedDB）
- 数据验证和提示
- 保存草稿功能

**主要组件：**
- `IxStepper` - 步骤指示
- `IxForm` - 表单
- `IxInput`, `IxNumberInput` - 数据输入
- `IxUpload` - 照片上传（支持相机拍照）
- `IxButton` - 保存/下一步按钮

#### 3.2.4 报告数据查看页 (`/task/review/:taskId`)
**功能要点：**
- 数据汇总展示
- 数据编辑和调整
- 数据验证状态
- 手动同步到云端
- 生成报告预览

**主要组件：**
- `IxCard` - 数据卡片
- `IxTable` - 数据表格
- `IxButton` - 同步/导出按钮
- `IxModal` - 数据编辑弹窗

### 3.3 管理员界面模块

## 七、设计规范

### 7.1 Siemens iX 组件使用规范
- 遵循 iX 设计系统的视觉规范
- 使用 iX 提供的图标库（`@siemens/ix-icons`）
- 保持组件使用的一致性
- 参考 [iX 官方文档](https://ix.siemens.io/docs/introduction)

### 7.2 颜色和主题
- 支持亮色/暗色主题切换（iX 内置支持）
- 使用 iX 语义化颜色（success/warning/error/info）

### 7.3 响应式断点
- 桌面端：>= 1024px（项目经理/管理员）
- 平板端：768px - 1023px（维护工程师）
- 移动端：< 768px（可选支持）
